# .impeccable.md

## Design Context

### Users
Developers who run multiple AI agents (e.g., Claude Code) simultaneously. They arrange multi-terminal grids with character image overlays on each panel to visually distinguish agents. Workflow context: long-duration monitoring, rapid panel switching, keyboard-centric workflow.

### Brand Personality
**Precise, Calm, Delightful**
- The feeling of deep focus in the quiet of dawn
- A balance between restrained refinement that aids concentration and the joy of character overlays
- Product sensibility that strips away the unnecessary, like Linear and Raycast

### Aesthetic Direction
- **Visual tone**: Industrial Utilitarian + Calm. A natural dark theme in forest green tones. Minimized UI chrome as a terminal-first tool. No glow effects, no glassmorphism, no gradient text.
- **Anti-references**: Cyan glow terminals (Warp/Hyper clones), VS Code complexity, glassmorphism
- **References**: Linear (restrained dark UI), Raycast (keyboard-centric interaction), Ghostty (terminal personality)

### Typography
Two stacks, strict separation:
- **UI chrome** (toolbar, status bar, dialogs, labels): System sans-serif
  ```
  -apple-system, BlinkMacSystemFont, 'Segoe UI', system-ui, sans-serif
  ```
- **Terminal content** (xterm.js): System monospace
  ```
  'SF Mono', 'Menlo', 'Monaco', 'Courier New', monospace
  ```

Panel header uses the UI sans-serif stack. Terminal content uses the monospace stack. Never mix them.

### Spacing
4px base grid (`--spacing: 0.25rem`). Compact everywhere.

- Grid gap: 6px (`gap-1.5`)
- Grid padding: 6px (`p-1.5`)
- Panel header: 10px horizontal, 3px vertical (`px-2.5 py-[3px]`)
- Panel header height: 24px (`h-6`)
- Toolbar height: 38px
- Status bar height: 24px
- Dialog max-width: 400px

### Border Radius
Minimal. Rounded rectangles, not pills.

- `--radius-sm`: 2px (inputs, small elements)
- `--radius-md`: 4px (panels, cards)
- `--radius-lg`: 6px (dialogs)
- `--radius-xl`: 8px (image previews)

### Motion
Minimal-functional. No decorative animation.

- State transitions: 200ms ease (border-color, box-shadow, ring-color)
- Micro-interactions: 150ms (color dot scale, button hover)
- Grid cell hover: 100ms (background, border-color, transform)
- `prefers-reduced-motion`: All durations â†’ 0.01ms

### Design Principles

1. **Terminal First** â€” Terminal content always takes priority. UI chrome is minimized, and all spacing defers to the terminal.
2. **Keyboard Native** â€” All core actions must be possible without a mouse. Visual feedback clearly communicates keyboard focus state.
3. **Quiet Until Active** â€” The default state is unobtrusive; presence is revealed through accent color only on interaction.
4. **Character as Identity** â€” Character overlays are a core feature and tgrid's differentiator. They visually convey agent presence without obscuring terminal content.
5. **Joyful Precision** â€” Even within a restrained interface, details bring delight. Smooth transitions, subtle character reactions, preset color personality.

## Implemented Patterns

### Active Panel State
Layered indicator system (`Panel.tsx`):
1. `ring-2` with panel color or accent â€” no layout shift
2. Border color matches ring color
3. Outer glow shadow at 20% opacity: `shadow-[0_0_14px_color-mix(in_srgb,var(--panel-color,var(--accent))_20%,transparent)]`
4. Header background: 8% accent tint via `color-mix(in srgb, panel-color 8%, surface)`
5. Inactive panels: plain border, no ring, no shadow, no tint

CSS custom property `--panel-color` set per-panel for preset colors, falls back to `--accent`.

### Preset Indicator (Color Dot)
Each panel header has a 10px (`size-2.5`) color dot on the right side:
- With preset: shows preset color, `hover:brightness-125`, `hover:scale-125`
- Without preset: `bg-muted-foreground/25`, `hover:bg-muted-foreground/50`
- Smooth `background-color` transition on preset change (150ms)
- Clicking opens PresetDropdown (radio-button style with images + color dots)
- Replaces the previous `...` menu button â€” smaller footprint, always shows preset color

### Character Overlay (`AgentOverlay`)
- Positioned bottom-right, 35% width, 70% height
- Bottom gradient fade to panel background (`before:` pseudo-element)
- Active panel: subtle scale-up (`scale-[1.02]`) with 300ms cubic-bezier transition
- Opacity: configurable per-state (default 15%, active 25%)

### Drag-and-Drop (Panel Swap)
- Panel headers are draggable for reordering
- Cursor: `grab` â†’ `grabbing` during drag
- Dragging panel: `opacity-50 scale-[0.97]` with smooth 200ms transition
- Drop target: green selection tint (`bg-selection/50`) with 150ms transition
- Assignments and PTY IDs swap together; xterm canvas state preserved via CSS `order`

### Dropdown Focus Style
Custom focus colors to avoid contrast issues with popover backgrounds:
- Dark theme: `focus:bg-white/[0.06]`
- Light theme: `focus:bg-black/[0.06]`
- Applied to all DropdownMenuItem, CheckboxItem, RadioItem, SubTrigger

### Toolbar
Three controls, visually unified with shared `TOOLBAR_BTN` style:
- Theme toggle (icon button: â˜¼/â˜¾)
- Presets dropdown (list + create + install pack)
- Grid resize dropdown (4x4 cell picker)
- All toolbar buttons: `active:translate-y-px` for tactile press feedback
- Style: `border border-input bg-secondary`, hover shows primary color

### Status Bar
- Shows active panel preset: color dot + preset name + "Panel N"
- Without preset: just "Panel N"
- Right side: panel count text + help dropdown (keyboard shortcuts)
- Text: `text-muted-foreground` for readability

### Grid Picker
- Full-screen overlay for initial grid size selection
- 4x4 cell grid with tactile hover (`hover:scale-105`, `active:scale-95`)
- Keyboard navigation: arrow keys to select, Enter to start
- Keyboard hint shown below Start button in ghost text
- Session restore: hold Shift on launch to show picker instead

### Grid Resize Dropdown
- Same 4x4 cell picker as GridPicker, in dropdown form
- Cells: `hover:scale-105`, `active:scale-95`
- Shows hover preview dimensions + agent count
- Handles PTY lifecycle on resize (kills removed, creates new)

### Preset Editor Dialog
- shadcn Dialog with `bg-background border-input`
- Image picker: 64px preview (`size-16`) with dashed placeholder
- Color swatches: 28px (`size-7`) with check mark indicator
- Shell override input (optional)
- Delete confirmation as nested dialog
- Labels: `text-secondary-foreground` for readability

### CSS Architecture
- Tailwind CSS v4 with cascade layers: `@layer theme, base, components, utilities`
- Theme switching via `data-theme` attribute (not `prefers-color-scheme`)
- Custom variant: `@custom-variant dark (&:is([data-theme="dark"] *))`
- shadcn/ui tokens mapped to tgrid CSS variables in `@theme inline`
- Token chain: `themes.css` â†’ `@theme inline` â†’ Tailwind utilities â†’ shadcn components

### Accessibility
- `prefers-reduced-motion`: all transitions/animations â†’ 0.01ms
- Panel containers: `role="region"` with `aria-label="Terminal N"`
- Color swatches: `role="radiogroup"` with `aria-checked`
- Preset dot: `aria-label="Assign preset"`, `aria-haspopup="menu"`
- Focus-visible rings on all interactive elements
- Scrollbar styling adapts to theme (light/dark)
